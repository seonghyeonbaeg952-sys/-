# 삼다수형 ‘시원한 연출’의 샘플 적용 근거

- 조사일: 2026-07-22
- 적용 범위: `/sample` 공개 화면만
- 제외 범위: 원본 `/` 및 기존 홈 히어로, 관리자 화면, 기존 모션 타이밍

## 2. 무엇을 적용하고 무엇을 배제하는가

| 원본에서 관찰한 원리 | 합창단 샘플 번역 | 채택 여부 | 이유 |
| --- | --- | --- | --- |
| 한 뷰포트에 한 메시지 | 소개·입단·공연·악보·정신·기록을 각각 독립 장면으로 유지 | 채택 | 기존 정보 구조를 바꾸지 않으면서 스캔 순서를 분명히 함 |
| 화면 폭 90% 이상을 쓰는 대형 시각 요소 | 사진 대신 `ONE VOICE` 배경 타이포를 섹션 폭으로 배치 | 채택 | 히어로 사진 직후 사진이 반복되는 문제를 없애면서 합창의 핵심 의미를 직접 전달 |
| 중앙 피사체 + 뒤의 대형 문자 | 중앙 피사체는 복제하지 않고 배경 문자의 넓은 스케일만 번역 | 부분 채택 | 제품 사이트의 물체 중심 문법을 합창단에 그대로 대입하지 않기 위함 |
| 섹션마다 강한 파랑/초록 전면 색 전환 | 흰색 바탕, 오렌지 큐, 악보에만 오버진 | 배제 | 사용자가 확정한 색 체계와 정보 가독성을 우선 |
| 제품 병을 반복하는 중앙 오브젝트 | 합창단 사진이나 임의 인물을 반복 배치 | 배제 | 청소년 사진 노출과 제품형 반복이 합창단 목적에 맞지 않음 |
| 강한 패럴랙스/스크롤 동기 이동 | 기존 유용한 모션은 유지, 새 배경은 고정하지 않음 | 배제 | W3C의 motion sensitivity 지침과 모바일 성능을 고려 |
| 물결 형태 섹션 경계 | 양끝 화살촉을 제거하고 중앙에만 대칭 M이 있는 직선 레일로 교체 | 채택 | 장식의 의미를 한 번에 인식하고, 다음 장면의 경계로 읽히게 함 |

## 3. 배경 타이포그래피와 전달력 근거

### 근거

- Nielsen Norman Group은 명확한 시각 위계가 사용자의 시선을 가장 중요한 요소로 이끌며, 위계는 색·대비, 크기, 그룹화로 만든다고 설명한다. 특히 큰 요소는 먼저 주목되므로 큰 요소를 1–2개로 제한하고, 중요하지 않은 요소에는 더 낮은 대비를 쓰는 것이 핵심이다.
- 같은 자료는 색과 대비 변형이 너무 많으면 위계가 약해지고, 공간만으로 그룹이 충분히 구분된다면 불필요한 컨테이너를 추가하지 말라고 권한다.
- GOV.UK Design System은 제목 계층을 일관되게 사용해 콘텐츠 구조를 명확히 하고, 본문 폭을 보통 75자 이내로 제한해 데스크톱 가독성을 유지한다. 또한 왼쪽 정렬은 긴 문장에서 눈의 복귀점을 일정하게 만든다.
- WCAG 1.4.3은 일반 텍스트 4.5:1, 큰 텍스트 3:1의 최소 대비를 요구한다. 의미를 전달하는 실제 제목과 본문은 이 기준을 지키고, `ONE VOICE`는 정보가 아닌 순수 배경 장식으로 낮은 대비를 사용한다.

### 샘플에 번역한 규칙

1. 첫 시선: 실제 HTML 제목 `서울모테트청소년합창단 소개`
2. 두 번째 시선: 설명과 정체성 정보
3. 세 번째 시선: `교육과 활동`의 네 항목
4. 배경 기억점: 화면 폭을 쓰는 `ONE VOICE`
5. 장면 종료: 중앙 M 직선 레일

따라서 배경 문자는 내용을 대신하지 않는다. 합창단의 가장 짧은 상징 문구만 크게 사용하고, 실제 정보는 높은 대비의 HTML 텍스트로 유지한다. 사진은 히어로와 이후 갤러리 장면에 남기고, 그 사이 소개 장면에서는 제거한다.

## 4. 카드 대신 배경을 쓰는 범위

NN/g는 카드를 ‘짧고 관련된 정보의 요약과 상세 링크’를 묶는 구성으로 정의하며, 카드가 많으면 목록보다 스캔이 어렵고 더 많은 공간을 사용한다고 설명한다. 따라서 샘플에서는 다음과 같이 나눈다.

- 유지: 입단·공연·공지처럼 클릭 가능한 요약 단위
- 완화: 합창단 소개처럼 하나의 연속된 서사
- 배경 전환: 감정과 브랜드 분위기를 전달하는 비상호작용 장면
- 금지: 단순히 현대적으로 보이기 위해 모든 문단을 카드로 감싸는 방식

소개 장면은 흰 배경과 여백으로 그룹을 구분한다. 왼쪽 소개문은 오렌지 세로선만 남기고 카드 배경·그림자·블러를 제거했다. 오른쪽 프로그램은 항목 비교가 필요하므로 카드 대신 얇은 상하 구분선과 2×2 목록 구조를 유지한다.

## 5. 접근성·성능 제한

- W3C WCAG 2.3.3 취지에 따라 새 패럴랙스나 뷰포트 대형 이동 모션을 추가하지 않는다.
- `prefers-reduced-motion: reduce`에서는 장식의 진입 모션을 중지한다.
- 히어로 직후 장면에서는 생성 이미지를 로드하지 않는다. 이로써 시각 중복과 추가 이미지 요청을 동시에 제거한다.
- 배경 타이포는 CSS 장식으로만 사용하며, 실제 제목과 본문을 대체하지 않는다.
- 오렌지 작은 텍스트는 `#C93A07`을 유지하고, `#FF601A`는 큰 글자·선·버튼·비텍스트 포인트에만 사용한다.

## 6. 적용 판단 체크

각 시각 효과는 아래 세 질문을 모두 통과할 때만 적용한다.

1. 사용자가 다음 정보 단위를 더 빨리 구분할 수 있는가?
2. 합창단의 정직한 음악·함께 부름·다음 세대라는 키워드와 연결되는가?
3. 모바일, reduced motion, 원본 경로를 해치지 않는가?

이번 적용에서 `ONE VOICE` 배경 타이포와 중앙 M 레일은 세 조건을 통과했다. 생성 사진 배경은 히어로와의 중복으로 1번을 통과하지 못해 제거했고, 강한 색면 전환과 신규 패럴랙스는 2번과 3번을 통과하지 못해 제외했다.

## 7. Chrome 스크롤 오류 분석과 수정

사용자 캡처에 나타난 잘림을 정지 화면이 아니라 실제 스크롤 상태 변화로 재현했다.

| 오류 | Chrome 실측 | 직접 원인 | 샘플 수정 |
| --- | --- | --- | --- |
| 장식이 선이 아니라 점·작은 V로 보임 | 청크 표면의 장식이 다음 섹션 흰 배경 아래에 놓여 일부 픽셀만 노출 | `home-flow-sample-chunk__surface::after`가 콘텐츠보다 낮은 레이어에 존재 | 청크마다 반복되던 장식을 제거하고, 소개 장면 끝의 M 레일 한 곳만 유지 |
| 합창단 정신 본문·버튼이 다음 장면에 잘림 | 패널 930px, 표시 영역 770px, 고정 top -176px. 다음 청크가 패널 종료 전에 상위 z-index로 진입 | 다음 청크 위치에 `coverDepth` 72–104px를 더한 조기 덮기 계산 | `/sample`만 트랙 자체의 자연 종료점으로 계산하고 청크 음수 마진 제거 |
| 390px 소개 내용 오른쪽 잘림 | 컨테이너 358px인데 자동 그리드 열이 476px로 계산 | 자식의 최소 콘텐츠 폭이 `auto` 트랙을 넓힘 | 모바일에서 `minmax(0, 1fr)`, 자식 `min-width: 0`, `width: 100%` 적용 |

수정 후 데스크톱에서 고정 패널이 끝나는 순간 패널 하단 595px, 다음 장면 상단 643px로 48px 간격이 남아 겹침이 사라졌다. 다음 스크롤 상태에서는 이전 패널 하단 175px, 다음 장면 상단 223px로 같은 간격을 유지했다. 모바일 `390 × 844`에서는 전체 문서 폭과 뷰포트 폭이 모두 390px로 일치했고, 고정 상태 속성은 비활성화되어 세로 흐름으로 전환됐다.
