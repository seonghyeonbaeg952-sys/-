- 교차 검증일: 2026-07-19
- 대상 문서: [`websites-a.md`](./websites-a.md), [`websites-b.md`](./websites-b.md), [`ux-trends-resources.md`](./ux-trends-resources.md)
- 목적: 실제 웹사이트 사례 200개와 UI·UX 학습 자료 126개의 반복 패턴을 하나의 판단 기준으로 정리하고, Figma 샘플 5개를 같은 축에서 비교한다.

## 1. 데이터 교차 검증 결과

### 검증 방식

- 실제 사이트 표는 URL의 scheme·host를 소문자로 통일하고 마지막 `/`를 제거한 값으로 중복을 검사했다.
- 도메인은 `www.`를 제거한 host 기준으로 교차 검사했다.
- 자료집은 `Y01–Y36`, `O01–O45`, `R01–R45` ID의 연속성, URL 유일성, 표 열 수를 검사했다.
- 모든 Markdown 표에서 행별 열 수를 비교하고, 각 조사 행에 유효한 Markdown 링크와 URL이 하나씩 있는지 확인했다.
- 같은 발행처의 서로 다른 공식 문서는 중복 URL로 보지 않았다. 예를 들어 W3C의 대비, 타깃 크기, 모션 문서는 서로 다른 검증 기준이다.

### 확인된 수량

|---|---:|---:|---:|---:|---|
| 실제 사이트 A | 100 | 100 | 100 | 100 | 일치 |
| 실제 사이트 B | 100 | 100 | 100 | 100 | 일치 |

YouTube는 모두 서로 다른 채널 URL이지만 같은 `youtube.com` host를 사용하므로 host 수는 1이다. UX 자료 전체의 37개 host는 발행처 반복을 허용한 값이며, 126개 URL 자체는 모두 고유하다.

## 3. SMYC 적용 우선순위

### P0 — 다섯 시안 모두 통과해야 하는 기본선

| 결정 | 이유 | 완료 기준 |
|---|---|---|
| 사용자 목적 3개를 Hero 직후에 분리 | 관객, 입단 희망자·보호자, 후원자·파트너의 과제가 다르다. | 현행 Hero는 보존하고, 바로 아래에서 `공연 보기`, `입단 안내`, `후원·문의`가 10초 안에 발견되며 텍스트 라벨이 있다. |
| 역할 기반 색 토큰 확정 | 현재 Navy·Gold·Ivory는 강점이지만 Gold 오용 시 대비가 약하다. | surface, text, accent, focus, success, danger 토큰과 허용 조합을 정의하고 WCAG 2.2 AA 대비를 확인한다. |
| 390px 모바일을 별도 구성 | 데스크톱 축소는 카드·Hero·책 모션의 조작성을 떨어뜨린다. | 가로 overflow 없음, 44px 타깃, 긴 제목·확대 텍스트 대응, 스와이프 없이 핵심 기능 사용 가능. |
| 모션 목적·중지·대체 상태 정의 | 기존 모션 자산은 좋지만 반복과 스크롤 가로채기는 피로와 접근성 문제를 만든다. | 각 모션에 목적, 시작 조건, 종료 조건, reduced-motion variant가 주석으로 적혀 있다. |
| 실제 데이터 예외 상태 포함 | CMS 데이터가 비거나 migration이 지연돼도 공개 홈은 깨지면 안 된다. | 공연 없음, 이미지 없음, 긴 공연명, 로딩, 오류, 모집 마감 상태의 컴포넌트가 있다. |

### P1 — 브랜드 차별화를 만드는 핵심

1. 현행 full-bleed Hero와 핵심 슬라이드 애니메이션을 다섯 안의 공통 고정 요소로 보존한다.
2. Hero 아래 정보면에는 흰색·웜 아이보리와 실제 오렌지 심볼을 사용하고, 프로그램 북의 읽기 경험은 B와 장문 모듈에서 집중 검증한다.
3. 악보·오선·페이지·조명이라는 기존 모션 자산을 `line-draw`, 짧은 page transition, spotlight의 세 계열로 정리한다.
4. 사진은 인물 중심 홍보보다 악보·지휘·연습 과정·공연 공간·포스터 아카이브를 우선한다.
5. 가치 문장 바로 아래에 공연, 교육, 기록, 후원 사용처 같은 검증 가능한 근거를 붙인다.

### P2 — 선택 시안에서만 검증할 실험

- View Transition으로 공연 카드와 상세 페이지의 이미지·제목 연속성을 만든다.
- 연혁 또는 공연 시즌에만 passive scroll progress를 사용한다.
- 선택형 오디오·화음 체험은 별도 진입점에서 사용자 재생 방식으로 실험한다.
- WebGL, 커스텀 커서, 스크롤재킹, 상시 마키, 자동 사운드는 기본 홈페이지에서 제외한다.

### 권장 조합

최종 홈페이지는 다섯 안 모두 **현행 full-bleed Hero를 동일하게 보존**한다. 첫 스크롤 아래는 A의 길찾기를 기본 후보로 두거나, 운영 우선순위에 따라 B 프로그램 북, C 상태 우선 입단, D 필터 아카이브, E 참여·후원 모듈을 조합한다. 다섯 Figma 샘플은 Hero를 다섯 방식으로 다시 그리는 실험이 아니라, 동일 Hero 아래에서 어느 과업 구조가 가장 잘 작동하는지 비교하는 실험이다.

## 5. 다섯 시안의 공통 Figma 검증 규격

- 프레임: Desktop `1440×2100`, Mobile `390×1688`, Reduced motion `390×1688`, Notes `520×1100`.
- 공통 과제: 첫 방문자가 합창단 성격 파악, 다음 공연 찾기, 입단 대상 확인, 후원 사용처 확인을 각각 수행한다.
- 공통 컴포넌트: Header, Hero, route card, 공연 카드, 공지 카드, score/spirit, archive, sponsor, support, Footer.
- 공통 variant: default, hover, focus-visible, active, disabled, loading, empty, error, reduced-motion.
- 공통 콘텐츠 스트레스 테스트: 2행이 넘는 한국어 공연명, 이미지 없음, 공연 없음, 모집 마감, 긴 장소명, 200% 텍스트 확대.
- 공통 평가: 5초 이해도, 첫 CTA 발견 시간, 모바일 오조작, 키보드 완주, 대비, 모션 목적, 실제 구현 가능성.

## 6. 선택 기준

각 콘셉트를 5점 척도로 평가하되 시각 취향만으로 고르지 않는다.

| 평가 항목 | 비중 |
|---|---:|
| 단체의 정신·격조 전달 | 25% |
| 공연·입단·후원 과제 성공 | 25% |
| 모바일 사용성 | 20% |
| 접근성·감소 모션 완성도 | 15% |
| 현재 React·CMS·모션 자산으로 구현 가능성 | 15% |

동점이면 데이터 손실 방지, 접근성, 모바일, 유지보수성 순으로 결정한다. 강한 시각 장면 하나보다 전체 사용자 흐름이 안정적인 시안을 우선한다.
