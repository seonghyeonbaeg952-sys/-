## 2. 채널 자체가 밝히는 관점

공식 채널 설명은 UI/UX Design, Design System, Figma, UX Strategy, Portfolio & Design Review를 핵심 범위로 제시한다. 또한 좋은 디자인의 목적을 화면 장식보다 사용자가 목표를 더 쉽고 빠르게 달성하도록 돕는 데 둔다. 이 관점은 최근 컨펌 영상의 제목과 컬러·여백·가독성 영상의 구체 설명에서 반복된다.

따라서 이 채널에서 가져올 핵심은 특정 유행 스타일이 아니라 다음의 순서다.

1. 사용자가 무엇을 먼저 봐야 하는지 정한다.
2. 컬러·폰트·여백으로 정보 계층을 만든다.
3. 버튼·인풋·셀렉트 같은 요소를 시스템으로 공통화한다.
4. 데스크톱 한 화면이 아니라 전체 페이지와 반응형 상태를 함께 본다.
5. 화려한 효과와 최신 트렌드는 위 네 조건을 통과한 뒤 제한적으로 사용한다.

## 4. 27개 표본에서 반복된 주제

### 4.1 “예쁜 한 화면”보다 사용 목적과 전체 흐름

최근 표본의 큰 비중은 시청자 화면을 직접 검토하는 컨펌이다. 특히 2GdjtbPSQTs의 공식 챕터는 GNB, Hero, 카테고리, 서비스, 상품, 브랜드 소개, Footer를 한 흐름으로 본다. 이는 SMYC 홈도 Hero만 바꾸는 것이 아니라 `공연 보기 → 상세`, `입단 안내 → 문의`, `후원 목적 → 연락`까지 이어서 평가해야 한다는 근거가 된다.

### 4.2 정보 계층이 세련미를 만든다

컬러 영상 4개, 폰트·여백 영상 5개, 가독성 영상 1개가 같은 결론을 반복한다. 강한 색과 큰 글자를 늘리는 것이 아니라 무엇을 먼저 보고 다음에 무엇을 읽을지 결정해야 한다. “모두 강하면 아무것도 강하지 않다”는 컬러 논리는 Hero, 일정, CTA, Footer 모두에 적용된다.

### 4.3 여백은 시선 이동을 줄이는 기능

AYbyBqkdbJk는 여백을 빈 공간이 아니라 불필요한 시선 처리를 줄이는 요소로 설명한다. 따라서 밝은 배경 시안은 콘텐츠를 성기게 흩어 놓는 것이 아니라, 관련 정보를 가까이 묶고 다른 그룹 사이를 충분히 벌려야 한다.

### 4.4 컴포넌트는 화면 복제가 아니라 운영 구조

F0kDoNaAxwY와 QCoLnJLEj7g는 컴포넌트 공통화, variant, property, slot, 버튼·Select·Input의 관계를 강조한다. 이는 Supabase CMS의 실제 데이터가 길어지거나 비어도 같은 구조를 유지해야 하는 SMYC에 직접 유효하다.

### 4.5 접근성과 가독성은 스타일 선택 전의 문턱

8sMNvinRLu0와 3WRndb8k2zU는 저시력, 밝은 야외, 4.5:1, 컬러 기준을 명시한다. 채널 설명은 WCAG 2.1을 언급하지만, 현재 기준은 [WCAG 2.2 Contrast Minimum](https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html)으로 교차 확인한다. 일반 텍스트 4.5:1과 큰 텍스트 3:1이라는 핵심 수치는 유지된다.

### 4.6 반응형은 별도 설계 대상

반응형·그리드 영상 3개가 장편 실습으로 반복된다. 다만 오래된 영상의 Figma UI와 세부 치수는 현재와 다를 수 있으므로, 채널에서는 반응형을 별도 과제로 다룬다는 원칙만 채택하고 구현 수치는 현재 코드·공식 Figma 문서로 확인한다.

### 4.7 AI와 유행은 기본기를 대체하지 않는다

RGFAGMRHzFU는 AI가 몇 장의 화면을 만드는 것과 디자이너의 지속 가능한 역할을 구분한다. Q5i9DkDV-FE는 Liquid Glass를 2026 트렌드 사례로 다루지만, [Nielsen Norman Group의 사용성 비평](https://www.nngroup.com/articles/liquid-glass/)은 투명도, 작은 타깃, 예측 가능성, 발견 가능성 문제를 지적한다. 트렌드로 관찰했다는 사실과 SMYC에 채택한다는 결론은 분리해야 한다.

### 직접 확인된 것과 확인되지 않은 것

- 채널 내부 `오렌지` 검색은 [포토샵 상품 이미지 영상](https://www.youtube.com/watch?v=jlPYghKIo7Q) 1개만 반환했고, UI 브랜드 컬러 원칙과는 무관했다.

1. **화이트 70–80%, Navy 텍스트 15–20%, 오렌지 5–10% 이내**를 시작점으로 삼되 면적 비율 자체를 규칙처럼 강제하지 않는다.

- 오렌지 그라디언트, 유리 효과, 그림자, 텍스처, 자동 모션을 한 요소에 중첩

- 포트폴리오 컨펌 사례는 취업용 화면에 대한 전문가 의견이며, 문화예술 비영리 사이트 사용자 연구를 대신하지 않는다.

### 해결되지 않음

- `@UXUIDesign`이 화이트+오렌지 팔레트를 공식적으로 추천한다는 근거는 찾지 못했다.

- 오렌지 solid CTA는 한 화면에 하나를 기본으로 하는가?
- White 위 모든 작은 텍스트·아이콘·focus 표시가 기준 대비를 통과하는가?
- 제목, 본문, 날짜, 캡션, 버튼의 계층이 색 없이도 구분되는가?
- 관련 정보는 가까이, 다른 그룹은 충분히 떨어져 있는가?
- Button·Input·Select·오류·완료 상태가 같은 컴포넌트 규칙을 쓰는가?
- 390px에서 데스크톱을 축소하지 않고 읽기 순서를 다시 배치했는가?
- 아이콘만 있는 버튼에 `aria-label`, tooltip, 44px 터치 영역이 있는가?
- 최신 효과를 제거해도 콘텐츠와 행동이 모두 이해되는가?
- 시안 리뷰에서 “예쁨” 외에 과제 성공, 가독성, 접근성, 유지보수를 함께 점수화했는가?
