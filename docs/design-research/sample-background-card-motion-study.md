## 2. 핵심 결론

### 2.1 카드는 없애는 것이 아니라 독립 단위에만 남긴다

Material Design은 카드를 하나의 주제에 관한 콘텐츠와 행동을 담는, 독립적으로 식별되고 스스로 설 수 있는 단위로 정의한다.

- 근거: https://m2.material.io/develop/web/components/cards
- 적용: 홈 히어로 바로 아래 `입단 안내`, `공연 일정`, `후원·문의`는 각각 다른 목적지와 행동을 가진다. 따라서 세 카드는 유지한다.
- 비적용: 합창단 소개, 교육 목적, 합창단 정신처럼 하나의 연속된 이야기를 잘게 나누어 모두 카드로 만들지 않는다.

### 2.2 연속 서사는 배경·여백·선으로 묶는다

지각 연구에서 근접성, 공통 영역, 연속성은 별도 테두리를 반복하지 않아도 항목을 그룹으로 인식하게 하는 강한 단서다.

- 공통 영역 연구: https://www.sciencedirect.com/science/article/pii/001002859290014S
- Gestalt 지각 연구 종합: https://pmc.ncbi.nlm.nih.gov/articles/PMC3482144/
- 근접성과 공통 색의 경쟁 효과: https://journals.sagepub.com/doi/10.1068/p270417

SMYC 샘플에서는 다음 순서로 그룹을 만든다.

1. 넓은 흰 배경으로 기본 읽기 흐름을 만든다.
2. 섹션 제목, 좌측 오선 레일, 오렌지 점을 같은 축에 정렬한다.
3. 한 이야기 안의 하위 항목은 간격과 얇은 선으로 묶는다.
4. 독립 행동이나 실제 물성 비유가 필요한 경우에만 카드·책·파일 오브젝트를 쓴다.

### 2.3 색은 면적보다 역할로 구분한다

- 흰색 `#FFFFFF`: 주 읽기 표면
- 오렌지 워시 `#FFF1E8`: 연속 내용 중 보조 설명 영역
- 소프트 오렌지 `#FFD0B8`: 경계, 오선, 비활성 장식
- 큐 오렌지 `#FF601A`: 주 행동, 현재 위치, 지휘 손, 진행 신호
- 딥 오렌지 `#D9470F`: 손 윤곽과 큰 그래픽. 흰색 대비가 약 4.32:1이라 작은 일반 텍스트에는 쓰지 않는다.
- 오렌지 잉크 `#C93A07`: 흰색 약 5.13:1, 오렌지 워시 약 4.64:1로 작은 강조 텍스트와 링크에 사용한다.
- 잉크 `#151515`: 본문과 제목
- 악보 오베르진 `#442044`: 악보·책 표지에만 제한

`#FF601A`는 흰색 위 약 3.03:1이므로 작은 본문색으로 쓰지 않는다. `#D9470F`도 4.5:1에 미달하므로 작은 오렌지 글자는 `#C93A07`로 분리하고, 밝은 오렌지는 면·굵은 선·큰 CTA에 집중한다. 기준은 WCAG 2.2의 일반 텍스트 4.5:1, 큰 텍스트 3:1이다.

- 근거: https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html

## 3. 카드 대 배경 의사결정표

| 홈 요소 | 판단 | 이유 | 샘플 표현 |
|---|---|---|---|
| 히어로 아래 3개 안내 | 카드 유지 | 목적지와 행동이 서로 독립 | 흰 카드, 소프트 오렌지 경계, 위치 유지 |
| 합창단 소개·교육 목적 | 배경형 | 하나의 연속 서사 | 흰 캔버스와 오렌지 워시, 얇은 분할선 |
| 입단 메시지 | 혼합 | 설명은 서사, CTA는 독립 행동 | 설명은 배경형, CTA만 버튼으로 분리 |
| 공연 프로그램 | 물성 오브젝트 유지 | 프로그램 북이라는 실제 매체 비유 | 종이 패널은 유지하되 베이지 그라데이션 제거 |
| 오선지·악보 | 물성 오브젝트 유지 | 합창단 고유 맥락을 가장 직접적으로 전달 | 악보 표지는 오베르진, 지휘 큐는 오렌지 |
| 합창단 정신 | 배경형 + 책 오브젝트 | 철학은 연속 서사, 책은 기록의 은유 | 섹션 배경은 흰색, 책 표지만 물성 유지 |
| 사진·영상·포스터 | 아카이브 오브젝트 유지 | 서로 다른 기록물을 한 묶음으로 탐색 | 파일철 형태, 오렌지 척추 |
| 후원 | 배경형 | 압박감 없는 읽기와 신뢰가 우선 | 흰 배경, 오렌지 워시 정보 면, 단일 CTA |

## 4. 히어로 아래 안내 카드의 배경 위치

카드 좌표는 바꾸지 않는다. 카드가 히어로의 마지막 장면 위에 걸쳐 있어 빠른 길을 제공하는 현재 공간 관계가 유효하기 때문이다.

문제는 카드 뒤 전체가 너무 일찍 흰색으로 덮여 히어로와 카드 사이 깊이가 사라진 점이었다. 해결은 다음과 같다.

- 카드 상단: 기존 히어로 이미지 위에 유지
- 본문 배경 시작: `.home-flow-body` 시작점에서 138px 아래
- 카드 위치·크기: 변경 없음
- 결과: 카드는 독립 행동으로 읽히고, 배경은 카드 중단부 아래에서 다음 본문을 받친다.

## 5. AI 오렌지 지휘 손 20프레임

### 5.1 생성 원칙

- 기존 5×4, 20포즈 시트를 동작 참조로 사용했다.
- 손은 `#FF601A` 중심, `#D9470F` 윤곽, `#FFD0B8` 하이라이트로 다시 생성했다.
- 배경은 누끼 전용 단색 크로마키로 생성한 뒤 알파로 변환했다.
- 원본 홈페이지가 쓰는 기존 프레임은 덮어쓰지 않았다.

### 5.2 누끼·정규화

- AI 원본에서 실제 알파 점유 행 4개와 각 행의 손 5개를 픽셀 점유율로 검출했다.
- 단순 4등분 시 생기던 이전 행 손목 조각을 제거했다.
- 새 손을 기존 각 프레임의 알파 바운딩 박스에 맞춰 크기와 기준선을 정규화했다.
- 20개 PNG 모두 네 모서리 알파가 0인지 확인했다.
- 녹색 우세 픽셀을 재검사한 결과 0개다.
- 총 프레임 용량은 약 1.37MB로, 기존 20프레임 약 1.76MB보다 작다.

### 5.3 모션 역할

손 모션은 계속 반복되는 배경 장식이 아니라 오선 흐름의 마침·지휘 큐로 사용한다. 기존 20프레임 지연 배열과 손 경로는 유지하고 이미지 자산만 샘플 전용으로 바꾼다.

- 공개 원본: `conductor-hand-frame-1..20.png`
- 샘플: `sample-conductor-hand-frame-1..20.png`
- 샘플 URL에서만 새 프레임을 선택한다.
- 트레일, 박자점, 심볼 리빌은 같은 오렌지 역할색을 쓴다.

## 6. 모션 접근성과 성능

W3C는 상호작용으로 시작되는 비필수 모션을 끌 수 있어야 하며, `prefers-reduced-motion`을 충분한 기법으로 제시한다.

- 근거: https://www.w3.org/WAI/WCAG21/Understanding/animation-from-interactions.html

MDN은 애니메이션에서 레이아웃을 다시 계산하는 속성보다 `transform`과 `opacity`가 합성 단계에서 처리될 가능성이 높다고 설명한다.

- 근거: https://developer.mozilla.org/en-US/docs/Web/Performance/Guides/Animation_performance_and_frame_rate

따라서 다음을 유지한다.

1. 프레임 이미지는 위치 계산 없이 같은 캔버스에서 opacity로 교체한다.
2. 손 경로 이동은 기존 transform 애니메이션을 유지한다.
3. 저모션 환경에서는 정적 최종 상태를 제공한다.
4. 애니메이션 때문에 버튼·텍스트의 DOM 순서나 클릭 위치를 바꾸지 않는다.

## 7. 구현 후 검증 기준

- `/sample` 외 경로에서 새 손 파일이 요청되지 않아야 한다.
- 20개 프레임이 모두 HTTP 200이며 `naturalWidth > 0`이어야 한다.
- 원본과 샘플의 애니메이션 이름·시간·easing 집합이 같아야 한다.
- 히어로, 안내 카드의 top/bottom/height가 변경 전과 같아야 한다.
- 안내 카드 뒤 배경 시작점만 138px 아래에 있어야 한다.
- 390px, 768px, 1440px에서 가로 overflow가 없어야 한다.
- `prefers-reduced-motion: reduce`에서 손 모션이 정적 대안으로 바뀌어야 한다.
- `pnpm lint`, `pnpm build`가 통과해야 한다.

## 8. 최종 디자인 원칙

이 샘플의 세련됨은 더 많은 효과가 아니라 역할이 겹치지 않는 데서 만든다.

- 카드는 독립 행동에만 쓴다.
- 배경은 연속된 이야기를 묶는다.
- 오렌지는 행동·진행·지휘 신호에 쓴다.
- 오베르진은 악보와 책 표지에만 쓴다.
- 손 애니메이션은 합창의 박자와 마침을 설명할 때만 나타난다.
- 홈 히어로와 기존 공간 구조는 유지한다.
