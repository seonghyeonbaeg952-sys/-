  2. 합창단의 사실과 가치에서 어떤 키워드를 추출하고, 이를 화면의 형태·재료·문장으로 어떻게 번역할 것인가?
  3. 흰색 화면에서 오렌지 포인트를 어디에, 어떤 강도로, 얼마나 써야 정보 계층과 브랜드 기억이 함께 살아나는가?

## 0. 결론 요약

1. **로고를 늘이지 않는다.** 마스터 로고는 그대로 두고, 심볼에서 반복되는 `중앙축·좌우대칭·직선 날개·안쪽으로 모이는 꺾임`이라는 구성 원리만 추출한다.
2. 장식의 주력안은 **가로 장식 `Symmetric M-Rail`**이다. 사용자가 그린 옆으로 누운 M 꺾임을 양끝에 거울대칭으로 두고, 두 직선을 길게 잇는다. 원본 심볼을 늘이지 않으면서 실제 심볼의 직선성과 좌우 균형을 살린다.
3. 장식은 카드 안을 채우지 않는다. 섹션 전환, 넓은 배경의 가장자리, 제목의 기준선, 악보 오브젝트의 외곽에만 둔다.
4. `M`, 오선, 손, 악보를 한 화면에서 동시에 주인공으로 만들지 않는다. **한 섹션에는 주 장식 문법 하나**만 사용한다.
5. 브랜드 핵심 키워드는 공식 근거가 강한 순서대로 `정직한 음악`, `순수한 합창`, `바른 이상`, `함께 부름`, `음악적 성장`, `지성·인성`, `다음 세대`, `기록과 무대`로 정리한다.
6. 오렌지는 “젊음”이나 “가장 눈에 띄는 색”이라는 막연한 심리 주장으로 쓰지 않는다. 연구상 시선은 색 하나보다 **대비, 위치, 텍스트·이미지의 의미, 과업**에 크게 좌우된다.
7. 정확한 대비 계산상 `#FF601A`는 흰색 위 `3.03:1`, `#D9470F`는 `4.32:1`이다. 따라서 둘 다 작은 일반 텍스트에는 쓰지 않는다. 작은 오렌지 텍스트는 흰색과 오렌지 워시에서 모두 4.5:1을 넘는 `#C93A07`로 분리한다.
8. 악보·책 표지의 오베르진 `#442044`는 오렌지와 역할이 겹치지 않는다. 오렌지는 지휘·행동·현재 상태, 오베르진은 악보의 물성과 깊이에만 사용한다.

### 2.3 SMYC 심볼에서 실제로 추출할 수 있는 형식 원리

현재 로컬 원본 `public/images/brand/smyc-symbol-transparent.png`를 기준으로 다음 특징을 확인했다.

1. 중심을 기준으로 좌우가 대응한다.
2. 수평으로 뻗은 날개와 안쪽으로 모이는 꺾임이 함께 있다.
3. 중앙의 수직 M과 아래의 수평 날개가 교차해 ‘지휘의 중심점’처럼 보인다.
4. 곡선보다 직선과 날카로운 방향 전환이 우세하다.
5. 작은 크기에서는 모든 세부보다 중심축과 좌우 날개의 실루엣이 먼저 읽힌다.

따라서 장식은 다음을 지킨다.

- 좌우 거울대칭을 기본값으로 한다.
- 1개의 중심 결절과 2개의 긴 수평 레일로 구성한다.
- 원본 로고의 글자 `SMY`나 전체 상징을 배경 패턴에 반복하지 않는다.
- 원본 로고 이미지를 CSS로 가로 확대하지 않는다.
- 모티프는 별도 SVG/CSS 선형 컴포넌트로 재구성하되, 로고로 오인되지 않도록 로고보다 단순하게 만든다.

### 2.4 장식 시안 A — `M-Wing Rail` / 중심 결절형

사용자가 직접 그린 가로 장식의 의도와 실제 심볼의 대칭성을 함께 반영한 안이다.

```text
───────────────╲╱╲╱───────────────
───────────────╱╲╱╲───────────────
```

위 표시는 구조 설명용이며 실제 로고를 재현한 것이 아니다.

- 전체 가로세로 비율: 데스크톱 `10:1~14:1`, 모바일 `5:1~8:1`
- 중심 결절 폭: 전체 폭의 `8~12%`
- 좌우 연장선: 각각 전체 폭의 `40~45%`
- 선 수: 기본 2개. 오선처럼 5개로 늘이지 않는다.
- 굵기: 장식 크기와 해상도에 따라 `1.5~3px`; 두 선은 같은 굵기
- 색: 기본 `#FFD0B8`, 활성 또는 섹션 마침점만 `#FF601A`
- 대칭: 중심축 기준 좌우 위치와 시각 무게를 맞춘다.
- 끝처리: 화면 바깥으로 자연스럽게 이어지거나 수직 마디선으로 닫는다.
- 모션: 등장할 때 전체를 계속 흔들지 않고, 중앙에서 양끝으로 1회 draw하거나 스크롤 진행을 표시한다.

적용 후보:

- 홈의 큰 서사 섹션 시작점
- 악보/프로그램 북의 장 제목 위 또는 아래
- Footer 직전의 마지막 마디
- 서브페이지 Hero 아래 첫 본문 전환

비적용:

- 홈 Hero 이미지 위
- 안내 카드 3개 내부
- 입단 입력폼의 각 필드
- 공연 날짜·장소·예매 버튼 주변

#### A-3 정제안 — `Symmetric M-Rail` / 최종 권장

사용자가 그린 장식은 옆으로 누운 M형 꺾임이 한쪽에 있고 두 직선이 길게 진행하는 구조였다. 이를 그대로 한쪽에만 두면 시각 무게가 치우치므로, 꺾임을 좌우 양끝에 거울대칭으로 배치한다.

- 양끝: 같은 폭과 각도의 M형 꺾임
- 중앙: 작은 원형 cue 하나만 허용
- 본체: 두 개의 긴 수평선
- 선 색: 위 선 `#FFD0B8`, 아래 또는 활성 선 `#FF601A`
- 장점: 사용자의 스케치와 실제 심볼의 대칭성을 함께 보존하고, 중앙에 큰 마크가 없어 제목과 경쟁하지 않는다.
- 판정: 세 안 중 최우선. 데스크톱 장식의 기본형으로 사용한다.

### 2.5 장식 시안 B — `M-Bracket`

M의 중앙축과 양쪽 날개를 세로 섹션 표식으로 단순화한 안이다.

- 폭이 좁은 데스크톱 여백에서 제목 블록을 묶는다.
- 정보가 많은 목록에서는 왼쪽 세로선과 섹션 번호만 남긴다.
- 모바일에서는 장식 전체를 축소하지 않고 `중앙 결절 + 짧은 선`으로 단순화한다.
- 현재의 긴 오선 레일과 동시에 쓰지 않는다.

판정: 보조안. 홈 전체의 주 장식으로 쓰면 사이드 레일과 경쟁하므로 일부 서브페이지에만 제한한다.

### 2.6 장식 시안 C — `Cadence Bar`

가로 M을 더 추상화해 `두 줄 + 중심 마침점 + 끝 마디선`만 남긴다.

- 일정, 공지, 연혁처럼 정보 밀도가 높은 구간에 적합하다.
- 장식보다 구조 구분선에 가깝다.
- M의 인상이 약하므로 브랜드 대표 장식으로는 A안보다 열위다.

판정: 기능형 보조안. A안과 같은 섹션에 겹치지 않는다.

### 2.7 장식 밀도 규칙

시각 복잡도와 색감은 첫인상에 빠르게 영향을 주고, 높은 시각 복잡도는 미적 평가와 부정적 상관을 보인다는 연구가 반복된다. 다만 최적값은 문화와 사용자에 따라 다르므로 고정 개수를 ‘과학적 기준’으로 주장하지 않는다.

- Reinecke 외, 450개 사이트·548명, 500ms 첫인상 연구: https://www.eecs.harvard.edu/~kgajos/papers/2013/reinecke13predicting.shtml
- UI 미학에서 시각 복잡도가 강한 예측 변수였던 연구: https://epub.uni-regensburg.de/43558/
- VisAWI의 네 축 `Simplicity, Diversity, Colorfulness, Craftsmanship`: https://www.uni-muenster.de/OWMS/uploads/drafts_thielsch/pdf/moshagen_2010.pdf

SMYC의 검증 가설:

1. 한 뷰포트에서 강한 장식 초점은 1개를 기본으로 한다.
2. 한 섹션에서 M 장식과 오선 장식이 둘 다 강하면 하나를 약화한다.
3. 장식이 본문 행간, 날짜, 버튼 외곽선에 겹치면 실패다.
4. 장식 제거 전후로 과업 성공률이 같고 브랜드 인지만 내려갈 때만 장식을 유지할 가치가 있다.

### 2.8 장식 접근성

W3C는 순수 장식 이미지를 CSS 배경으로 제공하거나 `alt=""`로 보조기술에서 제외하라고 안내한다. 장식이 스크린리더 출력에 들어가면 시각적 잡음이 음성 잡음으로 바뀐다.

- 근거: https://www.w3.org/WAI/tutorials/images/decorative/
- 구현 규칙:
  - 순수 장식 SVG는 `aria-hidden="true"`, `focusable="false"` 또는 CSS pseudo-element로 처리한다.
  - 장식 레이어는 `pointer-events: none`이다.
  - 장식 안에 실제 제목·날짜·상태 텍스트를 래스터로 넣지 않는다.
  - 장식이 주제 전환을 의미하면 시각 장식과 별개로 시맨틱 `section`, 제목, 필요하면 `hr`를 제공한다.

### 3.2 키워드 우선순위

홈에서의 순서는 다음으로 고정한다.

1. `공연 일정 / 입단 안내 / 후원·문의` — 방문자의 즉시 과업
2. `서울모테트청소년합창단` — 주체
3. `청소년아카데미 부설 / 2014 출범` — 관계와 사실
4. `음악적 성장 / 지성·인성 / 함께 부름` — 교육 방향
5. `순수한 합창 / 바른 이상` — 성인합창단에서 이어지는 철학
6. `기록 / 악보 / 무대` — 콘텐츠 탐색

감성 카피가 1~3을 밀어내면 키워드 계층이 실패한 것이다.

### 3.3 검색 키워드와 화면 문구의 관계

Google은 키워드 반복보다 사람에게 유용한 설명, 명확한 주제, 고유한 제목을 권한다. 제목을 같은 단어로 채우는 keyword stuffing은 피해야 한다.

- People-first content: https://developers.google.com/search/docs/fundamentals/creating-helpful-content
- 명확하고 고유한 title: https://developers.google.com/search/docs/appearance/title-link

권장 제목 예:

- `서울모테트청소년합창단 | 공연 일정과 입단 안내`
- `입단 안내 | 서울모테트청소년합창단`
- `공연·연주 기록 | 서울모테트청소년합창단`
- `합창단 정신 | 정직한 음악과 함께 부르는 공동체`

피해야 할 제목 예:

- `서울 합창단 청소년 합창단 서울모테트 어린이 합창단 모집`
- `최고의 청소년합창단`
- `세계적인 합창단`

## 4. 포인트 연구

### 4.1 ‘포인트’는 색 하나가 아니다

| 포인트 층 | 역할 | 수단 | SMYC 예시 |
|---|---|---|---|
| P0 행동 포인트 | 지금 눌러야 할 것 | 오렌지 면 + 명확한 동사 | `입단 안내`, `공연 보기` |
| P1 정보 포인트 | 지금 읽어야 할 것 | 큰 제목, 날짜, 위치, 굵기 | 공연 날짜, 모집 상태 |
| P2 브랜드 포인트 | 어디의 화면인지 기억 | M-Wing, 오렌지 심볼 | 섹션 전환 장식 |
| P3 분위기 포인트 | 깊이와 온도 | 워시 배경, 악보 오베르진 | 악보·프로그램 북 |

P0~P3를 모두 같은 진한 오렌지로 만들면 포인트가 아니라 소음이 된다.

### 4.2 시선 연구가 말하는 것

- KAIST의 아이콘 매트릭스 시선추적 연구는 특정 색 이름보다 전경과 배경의 높은 대비가 시각적 현저성을 강화한다고 보고한다: https://pure.kaist.ac.kr/en/publications/optimal-employment-of-color-attributes-to-achieve-saliency-in-ico/
- CHI 2023 UEyes 연구는 웹·데스크톱·모바일·포스터 UI에서 색 편향과 위치·콘텐츠 편향이 함께 작동하며, 텍스트가 이미지보다 더 자주 응시되는 경향을 보고한다: https://doi.org/10.1145/3544548.3581096
- 모바일 UI 193개, 참가자 30명 연구에서는 상단 좌측, 텍스트, 이미지의 편향이 강했고 색·크기 같은 bottom-up 요인의 영향은 상대적으로 작았다: https://arxiv.org/abs/2101.09176

따라서:

1. 오렌지는 의미 있는 라벨·동사·위치와 결합해야 한다.
2. 버튼을 오렌지로 칠했더라도 화면 아래에 숨겨져 있거나 문구가 모호하면 좋은 포인트가 아니다.
3. 제목, 날짜, 이미지가 이미 강한 곳에 오렌지 장식을 더 겹치지 않는다.

### 4.3 오렌지 색 역할과 대비 재계산

WCAG 상대 휘도 공식으로 현재 팔레트를 다시 계산했다.

| 색 | 흰색 대비 | 판정 | 허용 용도 |
|---|---:|---|---|
| `#FF601A` Cue Orange | `3.026:1` | 일반 작은 텍스트 실패, 큰 텍스트·비텍스트 경계 가능 | 큰 CTA 면, 굵은 선, 심볼, 큰 숫자 |
| `#D9470F` Deep Orange | `4.322:1` | 일반 작은 텍스트 4.5:1 미달 | 손 윤곽, 큰 글자, 장식 그래픽 |
| `#C93A07` Orange Ink | `5.132:1` | 흰색 통과 | 작은 강조 텍스트·링크 |
| `#C93A07` on `#FFF1E8` | `4.643:1` | 오렌지 워시에서도 통과 | 워시 위 작은 강조 텍스트 |
| `#151515` Ink | `18.261:1` | 통과 | 본문, 제목 |
| `#442044` Aubergine | `13.673:1` | 통과 | 악보 표지, 필요한 경우 흰 글자 배경 |

`#C93A07`도 `#FFD0B8` 위에서는 `3.658:1`이므로 소프트 오렌지 면 위 작은 글자에는 사용하지 않는다. 그 면에서는 `#151515` 또는 `#442044`를 사용한다.

- 대비 기준: https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html

### 4.4 오렌지 포인트의 배치 규칙

1. 페이지 전체의 주 CTA는 한 시점에 하나를 가장 강하게 보이게 한다.
2. 섹션 안에서는 `상태·행동·현재 위치` 중 하나만 Cue Orange로 지정한다.
3. 제목 전체를 오렌지로 칠하지 않고 첫 글자, 짧은 eyebrow, 번호, 결절 중 하나만 선택한다.
4. 긴 오렌지 선은 `#FFD0B8`; 중심점·활성 마디만 `#FF601A`다.
5. 본문 링크는 색만 쓰지 않고 밑줄, 화살표, 굵기 또는 포커스 링을 함께 쓴다.
6. 성공·경고·오류 상태를 모두 오렌지로 통합하지 않는다. 상태 의미는 기존 semantic 색과 텍스트 라벨을 유지한다.

### 4.5 악보 포인트

악보는 합창단에서 가장 강한 비인물 브랜드 자산이다. 하지만 ‘악보처럼 보이는 모든 곳’에 오선을 그으면 정체성이 희석된다.

- 악보 책 표지: 오베르진 `#442044`
- 실제 페이지: 흰색, 잉크, 매우 옅은 오렌지 마디 표식
- 현재 장·넘김 제어: Cue Orange
- 지휘 손: Cue Orange 계열 20프레임
- 카드 내부 장식 오선: 제거
- 배경 오선: 악보 섹션과 진행 레일에만 제한
- 악보 텍스트: 실제 HTML로 유지하고 장식 이미지에 굽지 않는다.

### 4.6 `@UXUIDesign` 영상과의 교차 검증

## 5. 카드 대 배경 재판정

W3C의 인지 접근성 지침은 긴 페이지를 논리적 섹션으로 나누고, 여백·선·배경색·카드 같은 시각 단서를 쓰되 아이콘과 단서가 너무 많아 인지 부하를 만들지 않도록 하라고 권한다.

- 근거: https://www.w3.org/WAI/WCAG2/supplemental/patterns/o2p03-page-structure/

| 콘텐츠 | 카드 | 배경·여백 | 장식 |
|---|---|---|---|
| 히어로 아래 3개 빠른 길 | 유지 | 카드 뒤 배경만 아래에서 받침 | 카드 내부 오선 제거, 번호만 유지 |
| 소개·교육 철학 | 제거 또는 최소 | 넓은 흰 면 + 워시 구획 | M-Wing 1회 |
| 공연 목록 | 항목 단위 유지 | 목록 전체는 흰 배경 | 날짜 축·마디선 |
| 악보 프로그램 | 책 물성 유지 | 주변은 비움 | 오베르진 표지 + 지휘 큐 |
| 입단 절차 | 단계가 독립일 때만 | 한 흐름으로 연결 | Cadence Bar 또는 번호 |
| 문의 폼 | 입력 그룹만 | 넓고 안정된 배경 | 장식 최소 |
| Footer | 카드 없음 | 흰색/워시 | 마지막 M-Wing 또는 마디선 1회 |

## 6. 모션 포인트 연구

모션은 기존 애니메이션을 유지하되 장식이 아니라 사건에 붙인다.

| 모션 | 사건 | 유지 조건 | 정적 대안 |
|---|---|---|---|
| Hero crossfade | 공연 장면 전환 | 텍스트가 안정되고 제어 가능 | 첫 장면 유지 |
| M-Wing draw | 새 장 시작 | 1회, 400~700ms, 읽기 방해 없음 | 완성 선 표시 |
| 오선 진행 | 섹션 진행 | 스크롤 위치를 설명 | 현재 마디만 표시 |
| 손 20프레임 | 지휘 마침·큐 | 마지막 구간에서만, 기존 타이밍 | 한 개 완성 프레임 |
| 카드 rise | 빠른 길 등장 | 처음 1회, 위치 변화 작게 | 즉시 표시 |

W3C는 비필수 상호작용 모션에 정지 또는 저모션 대안을 요구한다. MDN은 `transform`과 `opacity` 중심 애니메이션이 레이아웃 변경 속성보다 성능상 유리할 가능성이 높다고 설명한다.

- https://www.w3.org/WAI/WCAG21/Understanding/animation-from-interactions.html
- https://developer.mozilla.org/en-US/docs/Web/Performance/Guides/Animation_performance_and_frame_rate

## 8. 청소년 보호가 장식 방향에 미치는 영향

개인정보보호위원회는 아동·청소년에게 더 이해하기 쉽고 안전한 개인정보 보호 체계가 필요하다는 현장 의견과, 필요·최소한의 정보 수집 및 프라이버시 중심 설계를 강조한다.

- https://m.pipc.go.kr/np/cop/bbs/selectBoardArticle.do?bbsId=BS074&mCode=C020010000&nttId=11914
- https://www.pipc.go.kr/np/cop/bbs/selectBoardArticle.do?bbsId=BS074&mCode=C020010000&nttId=11818

따라서 화려한 장식에 단원 얼굴을 반복 사용하는 대신 다음을 우선한다.

1. 악보와 프로그램 북
2. 지휘 손과 손짓
3. 빈 무대, 좌석, 조명, 연습 공간
4. 얼굴 식별이 어려운 전체 장면
5. 날짜·장소·레퍼토리 같은 기록 메타데이터

장식 연구가 단순 미학이 아니라 청소년 노출을 줄이면서도 합창단다운 인상을 만드는 대안이 되어야 한다.

## 9. 구현 전·후 검증 계획

### 9.1 시각 검증

- 390px, 768px, 1440px에서 Symmetric M-Rail의 양끝이 같은 시각 무게를 갖고 콘텐츠 그리드와 맞는가?
- 가로 장식이 모바일에서 잘리거나 본문 위로 넘어오지 않는가?
- 한 뷰포트에 강한 주황 포인트가 경쟁하지 않는가?
- 카드 내부 오선이 제거되고 카드 위치·크기는 유지되는가?
- 악보 오베르진과 오렌지 CTA가 같은 역할처럼 보이지 않는가?

### 9.2 과업 검증

다음 과업을 각각 첫 클릭과 완료 여부로 확인한다.

1. 다음 공연 날짜와 장소 찾기
2. 현재 모집 상태 확인하기
3. 입단 문의로 이동하기
4. 성인합창단과 청소년합창단의 관계 설명 찾기
5. 과거 공연 기록 찾기

### 9.3 접근성 검증

- 장식 SVG가 접근성 트리에 노출되지 않는가?
- 키보드 포커스가 장식에 가지 않는가?
- `prefers-reduced-motion: reduce`에서 draw와 손 프레임이 정적으로 바뀌는가?
- 흰색·워시 위 작은 오렌지 텍스트가 `#C93A07` 이상 대비를 쓰는가?
- CJK 장문이 지나치게 긴 한 줄이 되지 않고 확대 시 재배치되는가?

W3C AAA 시각 표현 참고에서는 CJK 텍스트를 40 glyph 이하로 제한할 수 있는 메커니즘, 충분한 행간, 양쪽 정렬 회피를 설명한다. 이 수치를 모든 짧은 UI 문구의 고정 폭으로 오해하지 않고 장문 본문 검토 기준으로 사용한다.

- 근거: https://www.w3.org/WAI/WCAG22/Understanding/visual-presentation.html

### 9.4 실패 판정

다음 중 하나라도 발생하면 장식안을 줄이거나 재설계한다.

- 로고와 장식을 사용자가 같은 심볼의 왜곡으로 인식한다.
- 공연 날짜보다 M 장식이 먼저 읽힌다.
- 오렌지 버튼이 한 화면에 3개 이상 같은 강도로 경쟁한다.
- 장식 제거 시 과업은 쉬워지고 브랜드 인지도 차이가 거의 없다.
- 모바일에서 장식 때문에 가로 overflow가 생긴다.
- 장식 모션이 저모션 환경에서도 재생된다.

## 10. 최종 선택

현재 시안에 가장 적합한 조합은 다음이다.

1. **주 장식:** `Symmetric M-Rail` A-3 정제안
2. **보조 구조:** `Cadence Bar` C안
3. **배경:** 흰색 기본 + 오렌지 워시 구획
4. **주 행동:** `#FF601A` 면
5. **작은 오렌지 글자:** `#C93A07`
6. **악보 표지:** `#442044`
7. **본문:** `#151515`
8. **모션:** 기존 애니메이션 보존, M draw 1회와 지휘 손 마침만 의미 있게 사용
9. **카드:** 히어로 아래 3개와 독립 데이터 항목만 유지
10. **원본 보호:** 로고 파일과 `/sample` 밖 경로는 변경하지 않는다.

이 조합의 목표는 장식을 더 많이 넣는 것이 아니라, **SMYC만의 한 가지 직선 장식 문법을 만들고 나머지 화면을 조용하게 해 그 문법이 실제로 기억되게 하는 것**이다.
