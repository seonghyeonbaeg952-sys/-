# 200개 사이트 UI/UX 비교 평가 기준

- 확정일: 2026-07-19 (KST)
- 목적: 200개 사이트를 단순한 미감 순위가 아니라 서울모테트청소년합창단 홈페이지의 실제 과업에 비추어 같은 기준으로 비교한다.
- 대상 과업: `공연 찾기·상세 확인`, `합창단 정체성 이해`, `입단 정보 찾기`, `후원·문의`, `기록 탐색`.

## 1. 사이트별 필수 기록 항목

각 사이트마다 아래 내용을 모두 기록한다.

1. **UI 특징**: 색, 타이포그래피, 그리드·레이아웃, 이미지·재료감, 컴포넌트, 모션.
2. **UX 특징**: 첫 화면 우선순위, 내비게이션, 검색·필터, CTA, 상태 피드백, 긴 여정의 연결 방식.
3. **장점**: 실제 과업을 빠르고 명확하게 만드는 요소와 브랜드 고유성을 강화하는 요소.
4. **단점·위험**: 정보 은폐, 과도한 모션, 낮은 대비 가능성, 모바일 압축, 탐색 비용, 운영 난이도 등 관찰 가능한 위험.
5. **SMYC 적용성**: 그대로 채택할 원칙, 변형해서 쓸 원칙, 사용하지 않을 패턴.
6. **근거 상태**: 현재 공식 페이지 직접 확인, 일부 차단·검색 색인 보조, 또는 확인 불가.

## 2. 공통 점수표

각 축은 1–5점이다. 합계 30점은 해당 사이트 자체의 휴리스틱 평가이며, `SMYC 적합도` 1–5점은 별도로 기록한다. 모바일·키보드·저모션처럼 실제 상태를 확인하지 못한 축은 추정 점수를 넣지 않고 `N/A`로 남긴다. 이 경우 비교용 점수는 `취득점수 ÷ 확인한 축의 만점 × 30`으로 정규화하되, 확인 축 수를 함께 표시한다.

| 축 | 5점 | 3점 | 1점 |
|---|---|---|---|
| 핵심 과업 탐색 | 대표 과업과 상태가 첫 화면·1차 내비게이션에서 명확하고 다음 단계가 예측 가능 | 찾을 수 있으나 메뉴 탐색이나 스크롤이 필요 | 핵심 과업이 장식·캠페인 아래 숨거나 경로가 불명확 |
| 정보 위계 | 제목·메타·본문·CTA 순서가 색 없이도 즉시 구분 | 일부 구간은 명확하나 강한 요소가 경쟁 | 크기·색·카드가 모두 경쟁하거나 필요한 정보가 기본 상태에서 안 보임 |
| 브랜드 고유성 | 실제 콘텐츠·고유 자산·언어가 일관된 시스템을 형성 | 세련됐지만 흔한 패턴 의존 | 템플릿·효과가 기관 정체성을 대체 |
| 모바일 | 읽기 순서가 재구성되고 핵심 행동·터치·오버플로가 안정적 | 축소는 되지만 일부 밀도·동작이 부담 | 데스크톱 축소판, 가림·오버플로·과도한 고정 UI 위험 |
| 모션 유용성·제어 | 상태·관계·진행을 설명하고 자동 모션 제어·저모션 대안이 확인됨 | 절제됐으나 목적·제어가 일부 불명확 | 읽기를 지연하거나 자동·반복·스크롤 종속 모션이 탐색을 방해 |
| 접근성 신호 | 대비·키보드·명시적 라벨·정지 제어 등 여러 신호가 확인됨 | 일부 신호만 확인 | 낮은 대비·호버 의존·아이콘 단독 등 위험 신호가 뚜렷함 |

### SMYC 적합도

- **5**: 청소년 개인정보 보호, 합창단 정체성, 공연·입단·후원 과업, 현재 기술 구조에 큰 충돌 없이 적용 가능.
- **4**: 핵심 원칙은 유효하지만 색·콘텐츠·운영 방식의 조정이 필요.
- **3**: 한두 모듈의 참고에 적합하며 전체 방향으로 쓰기에는 비용·맥락 차이가 큼.
- **2**: 시각 영감은 있으나 과업·접근성·운영 구조와 충돌.
- **1**: 복제하지 않을 반례로서만 유효.

## 3. 우열을 가리는 방식

- 전체 1등 하나만 정하지 않는다. `공연 탐색`, `에디토리얼·아카이브`, `교육·입단`, `후원·커뮤니티`, `브랜드·모션`의 과업별 상위군을 먼저 정한다.
- 같은 점수에서는 **핵심 과업 탐색 → 접근성 신호 → 모바일 → 정보 위계 → 브랜드 고유성 → 모션** 순으로 우선한다.
- 강한 시각 효과가 있어도 핵심 정보가 늦거나 모션 제어가 없으면 상위로 올리지 않는다.
- 접근성 점수는 정식 WCAG 적합성 판정이 아니다. 공개 화면에서 확인한 신호에 대한 휴리스틱 점수이며, 확인하지 못한 기능을 있다고 추정하지 않는다.
- 총점만으로 순위를 정하지 않는다. 확인 축이 4개 미만인 사이트는 상위군 후보에서 제외하고 `관찰 보류`로 분류한다.
- 시즌·지역·로그인·쿠키 상태에 따라 화면이 달라질 수 있으므로 확인일과 접근 제한을 함께 기록한다.

## 4. 공통점·차이점 종합 방식

1. 200개를 업종군별로 비교해 반복 패턴과 업종 특유 패턴을 분리한다.
2. 동일 패턴이 과업 성공에 기여한 사례와 방해한 반례를 함께 묶는다.
3. 흰 바탕·오렌지 포인트, 실제 심볼, 편집형 구성, 기능적 모션에 대해 채택·조건부 채택·기각으로 분류한다.
4. 최종 Figma 5안에는 각 패턴의 출처 사이트, 얻은 장점, 피한 단점을 노트로 연결한다.
