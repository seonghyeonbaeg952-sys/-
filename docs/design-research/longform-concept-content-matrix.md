# 6,000px+ 장문 시안 A–E 사실 안전 콘텐츠·CTA 매트릭스

- 기준 시점: 2026-07-19 KST
- 범위: 기존 홈페이지 코드와 Figma 파일을 수정하지 않고, A–E 시안을 6,000px 이상 긴 브라우저 캡처형 화면으로 확장할 때 사용할 섹션·카피·CTA·상태·개인정보 규칙을 정의한다.
- 사실 기준: [`choir-factbook.md`](./choir-factbook.md)의 공식·기관·공연 원자료와 교차 검증 결과를 우선한다.
- UI·UX 기준: [`websites-comparison-synthesis.md`](./websites-comparison-synthesis.md), [`research-synthesis.md`](./research-synthesis.md), [`concept-directions-a.md`](./concept-directions-a.md), [`evidence-orange-motion.md`](./evidence-orange-motion.md), [`recursive-audit.md`](./recursive-audit.md)를 함께 적용한다.
- 비범위: 새로운 사실 추정, 가상 공연·후원 실적·후기 생성, 홈페이지 구현, Figma 프레임 수정.

## 1. 먼저 고정할 결론

1. 다섯 안 모두 현행 full-bleed Hero와 핵심 슬라이드 애니메이션을 같은 구성으로 보존한다. 비교 대상은 Hero가 아니라 첫 스크롤 아래의 정보 구조다.
2. 6,000px는 구현 시 고정 높이가 아니다. 아래 높이는 Figma 장문 캡처를 균형 있게 구성하기 위한 예상치이며, 실제 웹은 콘텐츠 길이에 따라 자연스럽게 늘어나야 한다.
3. 긴 화면은 반복 카드·반복 CTA·근거 없는 감성 문장으로 채우지 않는다. 정체성, 검증된 활동, 상태별 안내, 기록, 청소년 보호, 다음 행동을 한 번씩 분명하게 배치한다.
4. 2026 모집은 `2026-06-01~26 접수, 06-27 오디션` 일정이었고 기준 시점에는 종료됐다. 현재 시안에서 `모집 중`이나 활성 `지원서 작성하기`를 기본값으로 쓰면 안 된다. [Y08]
5. 확인된 미래 공연이 없으면 과거 공연을 미래처럼 보이게 하지 않는다. `공개된 예정 공연이 없습니다`, `공연 소식 확인`, `지난 공연 기록 보기` 중 데이터 상태에 맞는 문구를 쓴다.
6. 후원 계좌·금액·사용처·영수증·세제 혜택이 운영 확인되지 않은 상태에서는 결제를 약속하지 않는다. 기본 CTA는 `후원·문의 안내` 또는 `함께하는 방법 보기`다.
7. 청소년 단원은 사진 중심으로 홍보하지 않는다. 공개 동의와 `is_visible = true`가 함께 확인된 자료만 노출하며, 기본 시각 자산은 악보, 손, 지휘, 연습 공간, 빈 무대, 좌석, 조명, 포스터, 프로그램 북, 건축, 비식별 그룹 장면이다.
8. 실제 로고 심볼의 대표 오렌지 `#EB5307`은 흰색 위 대비가 약 3.64:1이므로 일반 크기 본문·링크 텍스트에 쓰지 않는다. 심볼, 큰 그래픽, 큰 글자, 선택 표식에 쓰고 작은 텍스트는 충분한 대비의 짙은 잉크색을 쓴다.

## 3. 6,000px+ 공통 화면 뼈대

### 장문 캡처 높이 예산

| 구간 | 예상 높이 | 역할 |
|---|---:|---|
| 공통 Hero | 900–1,050px | 현행 정체성과 슬라이드 제어를 그대로 보존 |
| 현재 상태·길찾기 | 320–520px | 공연, 입단, 후원·문의 중 현재 가능한 행동을 즉시 제시 |
| 시안의 주 과업 | 720–1,000px | A–E의 차이를 가장 분명하게 보여 주는 핵심 모듈 |
| 단체 정체성·교육 철학 | 700–900px | 공식 소개 문장과 실제 근거 연결 |
| 검증된 활동·기록 | 800–1,100px | 날짜·장소·자료 유형을 포함한 증거 블록 |
| 과정·관계·깊이 읽기 | 650–900px | 감성 카피가 아니라 사용자 이해를 돕는 상세 정보 |
| 청소년 보호·공개 원칙 | 520–700px | 사진·이름·문의 데이터의 공개 범위 설명 |
| 다음 행동·문의 | 600–800px | 현재 가능한 CTA만 제시 |
| Footer | 320–420px | 검증된 링크·정책·문의 경로 |
| **합계** | **약 6,230–7,500px** | 고정값이 아닌 장문 캡처 조립 기준 |

현재 `1440×2100`, `390×1688`, reduced-motion, notes 프레임 규격은 비교용 미리보기다. 6,000px+ 캡처는 이를 폐기하는 규격이 아니라, 선택된 시안을 실제 홈페이지 길이로 확장해 보는 별도 프레임이다.

### 장문 캡처 프레임 관례

- 데스크톱 비교 기준은 `1440×6800` Auto Layout 프레임으로 시작하고, 실제 콘텐츠가 길면 프레임을 늘린다. 6,000px를 맞추기 위해 섹션을 자르거나 빈 여백을 덧붙이지 않는다.
- 실제 구현에서는 섹션에 `height: 760px` 같은 고정값을 옮기지 않는다. 표의 높이는 캡처 구성 예상치이며 웹에서는 `min-height`도 꼭 필요한 시각 구간에만 쓴다.
- sticky header는 캡처에서 한 번만 보이게 한다. 긴 캡처 중간에 헤더를 반복 복제해 브라우저 스크린샷처럼 위장하지 않는다.
- 모바일은 `390×auto`로 별도 조립한다. 데스크톱의 6,000px 기준을 모바일에 그대로 강제하지 않고, 같은 정보·상태·CTA가 자연스러운 한 열 순서로 존재하는지만 확인한다.
- 캡처 하단은 마지막 실제 Footer에서 끝낸다. 빈 바닥, 가짜 쿠키 배너, 가상 브라우저 크롬으로 높이를 채우지 않는다.

### 공통 섹션 순서

1. **현행 Hero** — 같은 이미지, 같은 카피, 같은 pause/prev/next/dots.
2. **현재 상태** — 미래 공연 여부, 모집 상태, 문의 가능 여부를 데이터에서 계산.
3. **정체성 한 문장** — 청소년합창단의 소속, 창단일, 교육 방향.
4. **시안별 주 과업** — A 길찾기, B 프로그램 북, C 입단, D 기록, E 참여·후원.
5. **검증된 증거** — 공연 날짜·장소·프로그램, 공식 소개의 교육 목표.
6. **개인정보·청소년 보호** — 사진·이름·문의 정보의 공개 원칙.
7. **다음 행동** — 한 화면에 강한 주 CTA 하나, 보조 CTA는 낮은 위계.
8. **Footer** — 실제 라우트만 사용하고 주소·전화번호는 운영 확인 후 노출.

### 장문 화면의 반복 제한

- 같은 CTA 문구는 Hero 아래, 본문 중간, 마지막에 세 번 반복하지 않는다. 주 CTA는 상단 길찾기와 마지막 행동 중 최대 두 번 쓴다.
- 같은 모양의 둥근 카드가 3개 이상 이어지면 다음 섹션은 규칙선, 큰 숫자, 전체 폭 글, 원본 비율 자료 중 다른 문법으로 전환한다.
- 섹션 제목이 감성 문장뿐이면 바로 아래에 날짜, 장소, 출처, 상태, 대상, 다음 행동 중 최소 하나를 붙인다.
- 스크롤 진입 애니메이션은 콘텐츠 표시를 지연하지 않는다. 텍스트와 링크는 처음부터 DOM과 읽기 순서에 존재한다.

## 4. 공통 상태·CTA 매트릭스

### 4.1 모집 상태

| 상태 | 판정 조건 | 상태 라벨 | 본문 카피 | 주 CTA | 금지 |
|---|---|---|---|---|---|
| **open** | 접수 시작일 ≤ 오늘 ≤ 마감일이며 실제 지원 폼이 활성·검증됨 | `모집 중` | 확인된 대상, 마감일, 오디션일을 함께 표시 | `지원서 작성하기` → 실제 폼 | 폼 미검증 상태에서 활성 CTA, 근거 없는 소요 시간·답변 기한 |
| **upcoming** | 공식 미래 모집 일정은 있으나 접수가 아직 시작되지 않음 | `모집 예정` | 시작일과 공개된 안내만 표시 | `입단 안내 확인` → `/join` | 구현되지 않은 `알림 신청`, 예정일 추정 |
| **closed** | 마감일이 지났거나 공식 마감 공지가 있음 | `접수 종료` | `2026 모집은 종료되었습니다. 다음 모집 일정은 공식 안내에서 확인해 주세요.` | `입단 안내 확인` → `/join`; 보조 `모집 문의하기` → `/contact` | `현재 모집 중`, 활성 `지원서 작성하기` |
| **unknown** | 일정 필드가 없거나 데이터 오류·충돌 | `일정 확인 중` | `현재 모집 일정을 확인하고 있습니다.` | `입단 안내 확인` 또는 `모집 문의하기` | 빈 날짜를 현재 모집으로 해석, 과거 공고 재사용 |

**현재 샘플의 기본 상태:** `closed`. 접수 `2026-06-01~26`, 오디션 `2026-06-27`을 `종료된 2026 모집 일정`으로 표시한다. [Y08]

### 4.2 공연 상태

| 상태 | 판정 조건 | 안전 카피 | CTA |
|---|---|---|---|
| **future** | `is_visible = true`이고 공연 시작일이 미래이며 취소 상태가 아님 | 확인된 공연명·일자·시간·장소. 데이터가 없는 필드는 숨긴다. | `공연 상세 보기` → 실제 `/concerts/:concertId` |
| **none** | 공개된 미래 공연이 0개 | `현재 공개된 예정 공연이 없습니다. 지난 무대와 새 소식을 확인해 주세요.` | `지난 공연 기록 보기` → `/concerts?filter=past` |
| **loading/error** | 데이터 요청 중 또는 실패 | `공연 정보를 불러오는 중입니다.` / `공연 정보를 불러오지 못했습니다.` | `다시 시도`; 보조 `/concerts` |

- 흰색 위 일반 본문색으로 쓰지 않는다. 오렌지 심볼, 큰 연도, 큰 장 번호, active 밑줄, 진행선, 큰 면에 제한한다.
- 한 viewport에서 강한 오렌지 초점은 주 행동 하나 또는 현재 상태 하나다.
- 오렌지 상태는 색만으로 표현하지 않는다. 텍스트, 굵기, 아이콘, 밑줄, 형태 중 하나를 함께 쓴다.
- 링크와 작은 CTA 라벨은 검증된 짙은 잉크색을 사용한다. 접근성용 딥 오렌지는 대비 측정과 브랜드 승인 후 확정한다.

### 모션

| 보존·수정 가능 | 사용 목적 | 금지·대체 |
|---|---|---|
| Hero crossfade | 맥락 전환 | pause와 reduced-motion 정적 장면 없이 자동재생 금지 |
| 짧은 line-draw | 현재 위치·단계 변화 | 반복 장식선 애니메이션 금지 |
| 탭 indicator | 선택 상태의 이동 | 색만 바뀌는 상태 금지 |
| 목록→상세 shared transition | 같은 기록의 연속성 | 핵심 정보를 전환 뒤에 숨기지 않음 |
| letter fold | 사용처·문의 정보 공개처럼 원인과 결과가 있을 때 1회 | 자동 반복·정보를 접힌 상태로 유지 금지 |
| network pulse | 실제 선택·완료 피드백 때 1회 | 후원 효과를 암시하는 무한 pulse 금지 |

모든 안에서 wheel lock, scroll-jacking, 커스텀 커서 강제, 자동 소리, 끝없는 장식 pulse를 사용하지 않는다. `prefers-reduced-motion`에서는 큰 이동·scale·parallax를 정적 최종 상태 또는 짧은 opacity/color 전환으로 바꾼다.

## 12. 공통점·차이점·우열

### 다섯 안의 공통점

- 같은 현행 Hero와 유용한 슬라이드 애니메이션을 유지한다.
- Hero 아래는 흰색 또는 웜 아이보리 정보면, 짙은 잉크, 실제 오렌지 심볼을 쓴다.
- 소속, 2014-06-28 창단, 교육 방향, 검증된 공연 기록을 공통 사실 기반으로 삼는다.
- 공연·모집·후원 상태를 텍스트로 명확히 표시한다.
- 청소년 얼굴보다 활동 맥락과 원자료를 우선한다.
- 모든 CTA는 실제 라우트와 현재 운영 상태에 연결한다.
- 모바일 44px 터치 영역, 키보드 focus, reduced-motion, 로딩·오류·빈 상태를 포함한다.

### 핵심 차이

| 안 | 첫 질문 | 가장 강한 콘텐츠 | 가장 큰 운영 리스크 |
|---|---|---|---|
| **A 길찾기** | `나는 어디로 가야 하나?` | 현재 상태와 세 가지 과업 | 이야기와 기관 신뢰가 얕아질 수 있음 |
| **B 프로그램 북** | `이 합창단은 어떤 정신과 역사를 가졌나?` | 장문·철학·연표·원자료 | 행동 도달이 늦고 승인된 원문이 필요 |
| **C 상태 우선 입단** | `지금 지원할 수 있고 무엇을 확인해야 하나?` | 모집 상태·대상·일정·문의 | 시즌 종료 시 빈약, 최신 운영 정보 의존 |
| **D 필터 아카이브** | `어떤 기록을 언제·어디서 찾을 수 있나?` | 공연·포스터·프로그램·출처 | 결손 자료와 CMS 메타데이터 부담 |
| **E 참여·후원** | `어떻게 공동체와 연결될 수 있나?` | 미션·참여·협력·투명성 | 후원 정책·성과·개인정보 과장 위험 |

### 과업 적합도

점수는 외부 사이트나 미감의 절대 우열이 아니라 **현재 서울모테트청소년합창단의 과업과 근거 상태에 대한 적합도**다. 5가 가장 적합하다. `운영 부담 낮음`만 5가 관리하기 쉬움을 뜻한다.

| 평가축 | A | B | C | D | E | 1위 |
|---|---:|---:|---:|---:|---:|---|
| 공연 찾기·다음 행동 | **5** | 3 | 3 | 4 | 3 | A |
| 정체성·기관 신뢰 | 3 | **5** | 4 | **5** | **5** | B·D·E |
| 입단 상태·지원 탐색 | 4 | 3 | **5** | 2 | 4 | C |
| 기록·자료 탐색 | 3 | 4 | 3 | **5** | 4 | D |
| 참여·후원 문의 | 3 | 4 | 3 | 4 | **5** | E |
| 모바일 과업 명료성 | **5** | 4 | **5** | 4 | 4 | A·C |
| 현재 근거만으로 사실 안전성 | **5** | **5** | 4 | 4 | 3 | A·B |
| 운영 부담 낮음 | **5** | 4 | 4 | 2 | 3 | A |
| 개인정보 안전 설계 용이성 | 4 | **5** | 4 | **5** | 4 | B·D |

### 목적별 우열

1. **현재 홈페이지의 기본 뼈대:** A가 1위다. 첫 방문자에게 공연·입단·문의 경로를 가장 빠르게 제공하고, 현재 근거만으로도 안전하게 완성할 수 있다.
2. **기관 소개와 보호자 신뢰:** B가 1위다. 공식 소개·교육 철학·검증된 기록을 가장 자연스럽게 읽게 한다.
3. **모집 캠페인 기간:** C가 1위다. 단, 모집 일정과 실제 폼이 함께 활성화됐을 때만이다. 현재처럼 종료 상태에서는 A나 B 아래의 모듈로 쓰는 편이 낫다.
4. **기록 보존과 탐색:** D가 1위다. 출처·공개 동의·자료 유형을 지속 관리할 CMS 운영력이 전제된다.
5. **후원·협력 전환:** E가 1위다. 다만 사용처·결제·개인정보·영수증 정책이 승인되기 전에는 문의 중심 시안으로 제한한다.
6. **현재 사실 근거 기준의 즉시 제작 순위:** **A > B > C > D > E**. 이는 시각적 완성도 순위가 아니라 미확정 운영 정보에 덜 의존하는 순서다.

### 권장 결합

한 안의 미학을 전체 페이지에 강제하기보다 다음 조합이 가장 안전하다.

- **공통 Hero + A 길찾기**를 홈페이지 상단 기본 구조로 사용.
- B의 소개·교육 철학과 D의 검증된 기록을 중간 신뢰 블록으로 결합.
- C는 모집 상태가 open/upcoming일 때 상단으로 올리고, closed일 때는 입단 안내 모듈로 낮춤.
- E는 후원 운영 정책이 승인되기 전 `후원·협력 문의`로 제한하고, 승인 후 사용처·처리 흐름을 확장.

## 14. 시안별 최종 합격 기준

### 공통

- [ ] Hero가 다섯 안에서 같은 구조와 정보로 보존됐다.
- [ ] 장문 화면이 6,000px를 넘기기 위해 같은 카드·CTA·감성 문장을 복제하지 않았다.
- [ ] 모든 사실 문구에 factbook 근거 ID 또는 CMS 원자료 필드가 연결됐다.
- [ ] 미래 공연, 모집, 후원 상태가 현재 데이터와 일치한다.
- [ ] `#EB5307` 일반 크기 본문 텍스트 사용이 없다.
- [ ] 공개 인물 자료마다 동의와 `is_visible = true`가 확인됐다.
- [ ] 로딩·오류·빈 상태와 reduced-motion 화면이 있다.
- [ ] 390px에서 가로 overflow, 44px 미만 주 조작, scroll lock이 없다.

### A

- [ ] 공연·입단·후원·문의 경로를 10초 안에 찾을 수 있다.
- [ ] 미래 공연이 없을 때 과거 공연이 `다음 공연`으로 보이지 않는다.
- [ ] 길찾기 아래 정체성·교육·기록 근거가 충분하다.

### B

- [ ] 목차와 장 순서가 키보드·스크린리더 읽기 순서와 같다.
- [ ] 장문 카피가 공식 소개의 범위를 넘지 않는다.
- [ ] 가상 인용·프로그램 노트·후기를 만들지 않았다.

### C

- [ ] 기준 시점의 기본 상태가 `접수 종료`다.
- [ ] 지원 폼 CTA는 open 상태와 실제 폼 활성 조건을 모두 만족할 때만 보인다.
- [ ] 대상·회비·연습 장소·절차에 과거 공고와 현재 정보를 섞지 않았다.

### D

- [ ] 제7회 정보가 추정으로 채워지지 않았다.
- [ ] 성인 1989/1990 연혁과 청소년 2014 연혁이 별도 시간축이다.
- [ ] 모든 자료에 일자·유형·출처·공개 상태 메타데이터가 있다.

### E

- [ ] 후원 사용처·결제·영수증을 운영 확인 없이 약속하지 않았다.
- [ ] 출처 없는 impact 숫자와 가상 파트너 로고가 없다.
- [ ] 청소년 얼굴·후기를 감정적 전환 장치로 과사용하지 않았다.
